什麼是HTML

HTML(Hypertext Markup Language)並不是程式語言，它並沒有變數與流程控制等程式語言的基礎元素。它是標籤語言，作用在於告訴瀏覽器這份文件的結構。

HTML構成的要素："element"，一個element構成的要件通常是

  1. opening tag
  2. closing tag
  3. content

長得像這樣:

<tagName>content</tagName>

最後4.1.1. The html element

這邊就是介紹html這個元素，這個元素屬於文件元素之一，目前為止的規格中也只有html一種文件元素。接著我們一項一項看下去，

  1. Categories

    解釋該元素的類別。我們等一下看到的Content model裡面放的也是元素的類別， 不過指的是子元素的類別（這個元素的肚子裡面可以放什麼） Categories的相關介紹

    有趣的是這邊的類別寫的是None也就是無類別。嘿嘿！沒錯不是所有的元素都有類別，不過大部分的都有！ 而且一個元素可能同時屬於一種以上的類別。

  2. Contexts in which this element can be used

    基本上，這一項不用看，因為是non-normative description直接看下面的Content model

  3. Content model

    這邊寫的就是該元素裡面可以放什麼類別的元素。

    等等！這邊直接寫<head>和<body>不是寫類別耶！

    在這邊我們看到的又是個特例，這也告訴我們<html>裡面只能放一個<head>並且緊接著一個<body>元素喔！

  4. Tag omission in text/html

    這邊說的是該元素的opening/closing tag是否可忽略，請直接跳過。因為這項也是non-normative description

  5. Content attributes

    這邊講的是該元素可以設定的屬性，在<html>中我們可以設定Global attributes（全域屬性）和 manifest（已經準備廢棄的屬性可以忽略），其中最重要的是lang屬性，這個屬性標示這個網頁的主要語言， 在某些時候非常重要，例如：當你的使用者使用語音合成工具在讀你的網頁時！（視力不好的朋友也會上網的！） 目前這個網頁的lang屬性：<html lang="zh-cmn">

  6. Allowed ARIA role attribute values

    Allowed ARIA state and property attributes

    這兩項都是non normative請忽略

  7. DOM interface

    在學習JS之前請忽略

到這邊，我們看完了一個元素的規格，其實要看的地方也只有Categories、Content model 和Content attributes而已。

元素種類

下面規格書的3.2.4.2介紹html的7大類原素，有興趣的可以看看。但我們目前不會深入探討。（是不是鬆了一口氣？）

回想我一開始學習網頁的時候我腦中的元素只有三大類：

  1. 會佔據一行的元素Block elements

  2. 只會縮在一起的元素Inline elements

  3. 不會佔據一行又不會縮在一起的元素Inline block elements

舉例來說：我們剛剛在前面操作的<strong>就是inline element，包覆段落文字的<p>就是block element
